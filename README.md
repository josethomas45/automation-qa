# Automation QA – Selenium Web Automation

This repository contains **basic web automation scripts using Selenium WebDriver with JavaScript (Node.js)**.  
The project demonstrates how automated testing can simulate real user actions on a website such as opening pages, locating elements, entering data, clicking buttons, and extracting information.

The goal of this project is to **practice QA automation concepts and understand how Selenium interacts with web browsers.**

---

## 🚀 Technologies Used

- JavaScript (Node.js)
- Selenium WebDriver
- ChromeDriver
- CSS Selectors
- XPath

---

## ⚙️ Features

The automation scripts in this repository can:

- Launch a web browser automatically
- Navigate to a website
- Locate elements on the page
- Perform actions such as:
  - Typing text
  - Clicking buttons
- Scrape data from webpages
- Simulate real user interactions

## 🔄 Example Automation Flow

Typical Selenium automation flow used in this project:

1. Launch the Chrome browser  
2. Open a target website  
3. Locate webpage elements using CSS selectors or XPath  
4. Perform actions (click, type, extract data)  
5. Print or store the results  
6. Close the browser

## 🛠 Installation

Clone the repository:

```bash
git clone https://github.com/josethomas45/automation-qa.git
```

Move into the project folder:

```bash
cd automation-qa
```

Install dependencies:

```bash
npm install
```

---

## ▶️ Running the Automation Script

Run the script using Node.js:

```bash
node scrapeBooks.js
```

The script will:

- Launch Chrome
- Open the target website
- Extract book information
- Print the results in the console
